# Sets — Real Feed Implementation Plan

> **For agentic workers:** Inline execution — small scope, single file changes.

**Goal:** Replace hardcoded SETS data with an external JSON feed, enrich with SoundCloud URLs/genres/durations/dates, and connect ▶ play buttons to actual SoundCloud links.

**Architecture:** Static JSON file at `site/data/sets.json` fetched by `renderSets()`. SETS array in JS becomes a fetch fallback. Player ▶ opens SoundCloud in new tab.

**Tech Stack:** Vanilla JS, JSON, HTML

---

## File Structure

| File | Action | Description |
|------|--------|-------------|
| `site/data/sets.json` | Create | Structured set data: id, title, genre, duration, date, cover, url, live |
| `site/index.html` | Modify | Update `SETS` array → `fetch('data/sets.json')`, connect ▶ to SoundCloud |
